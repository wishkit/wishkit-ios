# Naming Guideline Reference

## Scope

Use for naming, API shape, and file naming consistency.

## Rules

1. Use lowerCamelCase for functions/properties.
2. Use UpperCamelCase for types.
3. Avoid underscores in Swift symbol names.
4. Use noun-based names for properties.
5. Use verb-based names for methods/actions.
6. Keep names specific enough to reveal intent.
7. Avoid banned terms if your project defines them (example: reserved architecture names).

## File Naming

1. File name should match the primary type in that file.
2. One enum per file.
3. One struct per file.
4. One class per file.
